International Domain Name Browser
=================================
     This sample demonstrates how to use the International Domain Name functionality provided in the System.Globalization.IdnMapping class with the new Windows Forms web browser control to create a web browser that accepts international domain names containing non-Latin characters.


Sample Language Implementations
===============================
     This sample is available in the following language implementations:
     C#
     Visual Basic


To build the sample using the command prompt:
=============================================
     1. Open the Command Prompt window and navigate to one of the language-specific subdirectories under the InternationalDomainName directory.
     2. Type msbuild InternationalDomainNameCS.sln or msbuild InternationalDomainNameVB.sln, depending on your choice of programming language.


To build the sample using Visual Studio:
=======================================
     1. Open Windows Explorer and navigate to one of the language-specific subdirectories under the InternationalDomainName directory.
     2. Double-click the icon for the .sln (solution) file to open the file in Visual Studio.
     3. In the Build menu, select Build Solution.
     The application will be built in the default \bin or \bin\Debug directory.


To run the sample:
=================
     1. Navigate to the directory that contains the new executable, using the command prompt or Windows Explorer.
     2. Type InternationalDomainName.exe at the command line, or double-click the icon for InternationalDomainName.exe to launch it from Windows Explorer. 

Demonstrates
============
     This sample demonstrates the following:
     System.Globalization.IdnMapping System.Windows.Forms.WebBrowser
